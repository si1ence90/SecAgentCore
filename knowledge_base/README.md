# SecAgent-Core 知识库

本目录包含 SecAgent-Core 的场景化知识库文件。这些文件以纯文本（.txt）格式存储，用于指导 Agent 在执行特定任务时的思考和规划。

## 知识库文件结构

每个知识库文件应遵循以下结构：

```
场景：[场景名称]

适用任务：
- [任务描述 1]
- [任务描述 2]
...

任务规划步骤：
1. [步骤 1]
   - [子步骤/详细说明]
2. [步骤 2]
   - [子步骤/详细说明]
...

工具使用指南：
- [工具名称 1]: [使用说明]
- [工具名称 2]: [使用说明]
...

注意事项：
- [注意事项 1]
- [注意事项 2]
...
```

### 字段说明：

- **场景**: 简要描述该知识库文件适用的主要场景。
- **适用任务**: 列出与该场景相关的用户任务描述关键词或短语。Agent 会根据用户输入与此部分的匹配度来判断是否应用该知识。
- **任务规划步骤**: 详细列出 Agent 在该场景下应遵循的执行步骤。Agent 会严格按照此处的规划进行任务拆解和工具调用。
- **工具使用指南**: 提供该场景下常用工具的简要说明和使用建议。
- **注意事项**: 列出在该场景下需要特别注意的安全事项或操作限制。

## 如何添加新的知识

1. 在 `knowledge_base/` 目录下创建一个新的 `.txt` 文件，例如 `my_new_scenario.txt`。
2. 按照上述"知识库文件结构"填写内容。
3. 确保"适用任务"部分包含用户可能输入的关键词，以便 Agent 能够正确匹配。

## 示例文件

- `port_scanning.txt`: 端口扫描任务的知识。
- `network_analysis.txt`: 网络分析任务的知识。
- `file_analysis.txt`: 文件分析任务的知识。

Agent 会在接收到用户任务后，首先搜索这些知识库文件。如果找到相关性高的知识，它将优先参考其中的"任务规划步骤"来指导其行为。
